================================================================================
RLHF (REINFORCEMENT LEARNING FROM HUMAN FEEDBACK) SYSTEM
================================================================================

COMPLETE IMPLEMENTATION âœ…

Data Collection:
  - Every assistant response has feedback buttons (0/0.5/1)
  - Feedback stored in feedback_responses table
  - Captures: query, response, score, chunks, model, timestamp

Data Stored:
  1. User Query (original question)
  2. Assistant Response (generated answer)
  3. Feedback Score (0.0=Bad, 0.5=Medium, 1.0=Good)
  4. Retrieved Chunks (context used)
  5. Model Used (phi4:14b or gpt-oss:20b)
  6. Retrieval Tier (vector/keyword/hybrid)
  7. Company ID
  8. Timestamp
  9. Review Status (pending/reviewed/resolved)

Review Workflow:
  - GET /api/feedback/bad - All bad responses
  - GET /api/feedback/medium - All medium responses
  - GET /api/feedback/needs-review - Unreviewed feedback
  - PATCH /api/feedback/{id}/review - Mark as reviewed

Analytics:
  - Summary by company (avg score, good/medium/bad counts)
  - Summary by model (compare phi4:14b vs gpt-oss:20b)
  - Database views for quick queries

Future Use:
  1. Prompt engineering (analyze bad responses)
  2. Retrieval tuning (check if bad chunks caused errors)
  3. Model comparison (which model performs better)
  4. Fine-tuning dataset (good/bad pairs)
  5. Pattern detection (identify failing query types)
